**Data and integration engineer.** I build the Python and SQL systems that move enterprise data between platforms that were never designed to talk to each other, and I work directly with the people who depend on the output.

Six years across enterprise SAP S/4HANA migrations, finance analytics, and automation at a Fortune 500 industrial technology company.

### What that looks like in practice

- **Multi-source integration pipelines.** Python ingestion, SQL transformations on a medallion model (bronze, silver, gold), plus a dedicated observability schema so the output is auditable. One of these cut a finance reporting cycle from 16-17 days to under a day, pulling from four separate source systems.
- **Automation that removes recurring manual work.** One replaced an 8-10 hour preparation and validation cycle with a 30-minute run.
- **Applied AI on real work, not demos.** A two-agent workflow: when a file arrives in a new format, one agent generates the table-creation SQL and a second reviews it before it is applied. The reviewer is the whole point. Nothing gets to touch the schema on a single model's say-so.
- **Enterprise data migration** across Mexico, the US, Canada, and EMEA, delivered on site in Milwaukee, Toronto, Edmonton, Brampton, and Bratislava.
